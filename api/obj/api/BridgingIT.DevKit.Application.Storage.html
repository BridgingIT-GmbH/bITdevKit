Interfaces

IDocumentStoreClient<T>

Defines the public API used by application code to query and mutate typed documents.

IDocumentStoreProvider

Defines the backend contract implemented by document-store providers.

IFileEventProcessor

Defines the contract for processing FileEvent instances in the FileMonitoring system. Processors are chained sequentially per location to handle detected file events.

IFileEventStore

Defines the contract for storing and retrieving FileEvent instances in the FileMonitoring system. Acts as a bridge between the domain layer and persistence layer, abstracting storage details.

IFileMonitoringService

Defines the contract for the FileMonitoringService, which orchestrates file monitoring across multiple locations. Provides methods for starting, stopping, scanning, and managing the monitoring process, along with events for external integration.

IFileStorageBehavior

Defines a behavior/decorator contract for extending IFileStorageProvider functionality, such as logging, caching, or retrying operations, while preserving the Result pattern.

IFileStorageProvider

Defines a contract for abstracting file system interactions across various storage types, including traditional file shares and cloud storage systems like Azure Blob Storage. Hierarchical structure is mimicked via path naming (e.g., blob names like "folder/subfolder/file.txt"). Containers or shares are configured during provider initialization. All operations return a Result to handle success/failure with messages and typed errors.

IFileStorageProviderFactory

Factory interface for creating and configuring file storage providers. Supports registration, creation, and behavior decoration of storage providers.

IFileStorageTreeRenderer
ILocationHandler

IMonitoringBehavior

Defines the contract for global behaviors that observe scan operations in the FileMonitoring system. Provides hooks for monitoring scan lifecycle events across all locations.

IProcessorBehavior

Defines the contract for behaviors that enhance IFileEventProcessor execution. Behaviors execute before and after processing, providing features like logging or retry logic.
