Examples

var services = new ServiceCollection();
services.AddRequester()
    .AddHandlers(new[] { "^System\\..*" })
    .WithBehavior<ValidationBehavior<,>>();
var provider = services.BuildServiceProvider();

Remarks

This class provides a fluent API for registering handlers, behaviors, and providers for the Requester system. It scans assemblies for handlers and validators, caches handler types in a shared IHandlerCache, and registers services with the DI container. Uses HandlerCacheFactory to ensure a single shared cache.

The RequesterBuilder instance for fluent chaining.

Examples

services.AddRequester()
    .AddGenericHandler(typeof(GenericDataProcessor<>), typeof(ProcessDataRequest<>), new[] { typeof(UserData) });

Remarks

Registers closed generic handlers in the DI container and the shared IHandlerCache. Also registers nested validators for the closed request types if they implement FluentValidation.IValidator<>.

Exceptions

ArgumentException

Thrown when genericHandlerType or genericRequestType is not an open generic type definition, or if the number of type arguments does not match the generic parameters.

ArgumentNullException

Thrown when genericHandlerType, genericRequestType, or typeArguments is null or empty.

AddGenericHandlers(IEnumerable<string>)

Automatically discovers and registers generic request handlers by scanning assemblies for open generic handlers and their corresponding request types, based on generic constraints.

public RequesterBuilder AddGenericHandlers(IEnumerable<string> blacklistPatterns = null)

Parameters

blacklistPatterns IEnumerable<string>

Optional regex patterns to exclude assemblies (e.g., "^System\..*"). Defaults to ApplicationDependencies.

Returns

RequesterBuilder

The RequesterBuilder instance for fluent chaining.

Examples

services.AddRequester()
    .AddGenericHandlers(new[] { "^System\\..*" });

Remarks

Discovers types implementing IRequestHandler<TRequest, TValue> where TRequest is a generic type definition. Registers closed handlers for concrete types satisfying the generic constraints, along with nested validators.

Exceptions

InvalidOperationException

Thrown when a generic handler has an invalid number of generic parameters or no valid type arguments are found.

Adds the handler to the DI container and the shared IHandlerCache. Also registers any nested Validator class implementing FluentValidation.IValidator<> for the request type.
