Remarks

This class encapsulates all settings required to configure JWT Bearer authentication in an ASP.NET Core application. It includes token validation parameters, issuer/audience configuration, and metadata requirements.

Typically, these options are populated from application configuration (appsettings.json) under the "Authentication" section and passed to the authentication service collection extensions.

Example configuration: 

{
  "Authentication": {
    "Authority": "https://auth.example.com",
    "ValidIssuer": "https://auth.example.com",
    "ValidAudience": "my-api",
    "ValidateIssuer": true,
    "ValidateAudience": true,
    "ValidateLifetime": true,
    "ValidateSigningKey": true,
    "RequireHttpsMetadata": true,
    "SaveToken": true,
    "RequireSignedTokens": true,
    "ClockSkew": "00:05:00"
  }
}

Properties

Authority

Gets or sets the authority URL where the authorization server is located.

public string Authority { get; set; }

Property Value

string

The base URL of the authorization server (e.g., "https://login.microsoftonline.com/common"). This is required and used to fetch OpenID Connect metadata and signing keys.

ClockSkew

Gets or sets the clock skew tolerance for token lifetime validation.

public TimeSpan ClockSkew { get; set; }

Property Value

TimeSpan

A timespan representing the allowed difference between server times during token validation. Accounts for minor clock differences between the token issuer and validator. Defaults to 5 minutes.

RequireHttpsMetadata

Gets or sets a value indicating whether HTTPS metadata is required.

public bool RequireHttpsMetadata { get; set; }

Property Value

bool

True to require the authority URL to use HTTPS and reject HTTP metadata endpoints; false to allow HTTP. Defaults to false. Should be set to true in production environments for security.

RequireSignedTokens

Gets or sets a value indicating whether tokens must be cryptographically signed.

public bool RequireSignedTokens { get; set; }

Property Value

bool

True to reject unsigned tokens; false to accept unsigned tokens. Defaults to false. Should be set to true in production for security.

SaveToken

Gets or sets a value indicating whether the access token should be stored in the HTTP context.

public bool SaveToken { get; set; }

Property Value

bool

True to save the token in the authentication properties, making it accessible via the HTTP context; false to discard the token after validation. Useful for accessing the token in application code. Defaults to false.

SigningKey

Gets or sets the signing key used for symmetric token validation.

public string SigningKey { get; set; }

Property Value

string

A base64-encoded or plaintext key used to validate token signatures when using symmetric algorithms. Only used if ValidateSigningKey is true and the authority is not used for key retrieval. Typically used with HS256 (HMAC-SHA256) algorithms. Defaults to null (keys fetched from authority's metadata endpoint).

ValidAudience

Gets or sets the valid audience for the API.

public string ValidAudience { get; set; }

Property Value

string

The expected audience claim in the token. Tokens with a different audience will be rejected if ValidateAudience is true. Example: "my-api", "https://api.example.com"

ValidIssuer

Gets or sets the valid issuer URL that tokens should be issued from.

public string ValidIssuer { get; set; }

Property Value

string

The expected issuer URL. Tokens with a different issuer will be rejected if ValidateIssuer is true. Example: "https://auth.example.com"

ValidateAudience

Gets or sets a value indicating whether to validate the token audience.

public bool ValidateAudience { get; set; }

Property Value

bool

True to validate that the token audience matches ValidAudience; false to skip validation. Defaults to false.

ValidateIssuer

Gets or sets a value indicating whether to validate the token issuer.

public bool ValidateIssuer { get; set; }

Property Value

bool

True to validate that the token issuer matches ValidIssuer; false to skip validation. Defaults to false.

ValidateLifetime

Gets or sets a value indicating whether to validate the token lifetime (expiration).

public bool ValidateLifetime { get; set; }

Property Value

bool

True to reject expired tokens; false to accept tokens regardless of expiration. Defaults to false. Should generally be set to true for security.

ValidateSigningKey

Gets or sets a value indicating whether to validate the token signing key.

public bool ValidateSigningKey { get; set; }

Property Value

bool

True to validate that the token is signed with a known and trusted key; false to skip signing key validation. Defaults to false. Should be set to true for security.
