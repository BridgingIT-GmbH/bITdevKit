Remarks

Defines what origins, methods, headers, and credentials are allowed for cross-origin requests. Null values for boolean properties are treated as false (feature not enabled).

Configure policies based on your security requirements: - Production: Use specific AllowedOrigins with AllowCredentials = true - Development: Use AllowAnyOrigin = true (without credentials) or specific localhost origins - Public APIs: Use AllowAnyOrigin = true (without credentials)

Remarks

When true, allows any header in cross-origin requests. Overrides AllowedHeaders when true. Simplifies configuration but reduces control over what headers clients can send. Default: false (null treated as false)

AllowAnyMethod

Gets or sets a value indicating whether any HTTP method is allowed.

public bool? AllowAnyMethod { get; set; }

Property Value

bool?

Remarks

When true, allows all HTTP methods (GET, POST, PUT, DELETE, PATCH, etc.). Overrides AllowedMethods when true. Simplifies configuration for APIs that support multiple methods. Default: false (null treated as false)

AllowAnyOrigin

Gets or sets a value indicating whether any origin is allowed.

public bool? AllowAnyOrigin { get; set; }

Property Value

bool?

Remarks

When true, allows cross-origin requests from any origin (*).

WARNING: Cannot be used with AllowCredentials = true (violates CORS specification).

SECURITY: Only use in development or for public APIs that don't require authentication. For production applications with user data, always specify exact origins in AllowedOrigins.

Overrides AllowedOrigins when true. Default: false (null treated as false)

AllowCredentials

Gets or sets a value indicating whether credentials (cookies, authorization headers) are allowed.

public bool? AllowCredentials { get; set; }

Property Value

bool?

Remarks

When true, allows credentials to be sent in cross-origin requests.

WARNING: Cannot be used with AllowAnyOrigin = true (violates CORS specification). When using credentials, AllowedOrigins must specify exact origins (no wildcards except subdomains).

Use Cases: - Frontend applications that send authentication tokens or cookies - APIs that require user authentication for cross-origin requests

Default: false (null treated as false)

AllowWildcardSubdomains

Gets or sets a value indicating whether wildcard subdomain matching is enabled.

public bool? AllowWildcardSubdomains { get; set; }

Property Value

bool?

Remarks

When true, origins in AllowedOrigins are treated as base domains allowing any subdomain.

Example: "https://example.com" allows: - "https://api.example.com" - "https://app.example.com" - "https://admin.example.com"

Requires AllowedOrigins to be specified (does not work with AllowAnyOrigin). Compatible with AllowCredentials = true.

Default: false (null treated as false)

AllowedHeaders

Gets or sets the array of allowed request headers (e.g., "Content-Type", "Authorization").

public string[] AllowedHeaders { get; set; }

Property Value

string[]

Remarks

Specifies which headers can be sent in cross-origin requests. Cannot be used when AllowAnyHeader is true. Simple headers (Accept, Accept-Language, Content-Language, Content-Type) are always allowed.

Example: ["Content-Type", "Authorization", "X-Custom-Header"]

AllowedMethods

Gets or sets the array of allowed HTTP methods (e.g., "GET", "POST", "PUT").

public string[] AllowedMethods { get; set; }

Property Value

string[]

Remarks

Specifies which HTTP methods are allowed for cross-origin requests. Cannot be used when AllowAnyMethod is true. If not specified and AllowAnyMethod is false, defaults to GET only.

Common values: "GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"

Example: ["GET", "POST", "PUT", "DELETE"]

AllowedOrigins

Gets or sets the array of allowed origins (e.g., "https://example.com").

public string[] AllowedOrigins { get; set; }

Property Value

string[]

Remarks

Specifies which origins are allowed to make cross-origin requests. Origins must include scheme, host, and optionally port (e.g., "https://localhost:5001"). Do not include trailing slashes. Cannot be used when AllowAnyOrigin is true. When AllowWildcardSubdomains is true, specify base domain (e.g., "https://example.com" allows https://*.example.com).

Example: ["https://example.com", "https://app.example.com", "https://localhost:3000"]

ExposeHeaders

Gets or sets the array of headers exposed to the browser (e.g., "X-Total-Count").

public string[] ExposeHeaders { get; set; }

Property Value

string[]

Remarks

Specifies which response headers the browser should expose to JavaScript. By default, only simple response headers are exposed (Cache-Control, Content-Language, Content-Type, Expires, Last-Modified, Pragma). Use this to expose custom headers like pagination info or request IDs.

Example: ["X-Total-Count", "X-Page-Number", "X-Request-Id"]

PreflightMaxAgeSeconds

Gets or sets the preflight cache duration in seconds.

public int? PreflightMaxAgeSeconds { get; set; }

Property Value

int?

Remarks

Specifies how long (in seconds) the browser can cache the preflight OPTIONS request result. Reduces preflight requests for subsequent cross-origin requests from the same origin.

If not specified, browser default is used (typically 5 seconds). Recommended: 600-3600 seconds for production to reduce overhead.

Example: 3600 (1 hour)
