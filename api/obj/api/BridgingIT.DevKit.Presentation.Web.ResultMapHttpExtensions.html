Map<TSuccess, TUnauthorized, TBadRequest, TProblem>(Result, Func<TSuccess>, ILogger)

Maps a non-generic Result struct to HTTP results for operations, handling success, unauthorized, not found, bad request, validation, conflict, rule violations, and other error cases in a flexible manner.

public static Results<TSuccess, TUnauthorized, TBadRequest, TProblem> Map<TSuccess, TUnauthorized, TBadRequest, TProblem>(Result result, Func<TSuccess> successFunc, ILogger logger = null) where TSuccess : IResult where TUnauthorized : IResult where TBadRequest : IResult where TProblem : IResult

Parameters

result Result

The Result struct containing the operation outcome.

successFunc Func<TSuccess>

A function to generate the success HTTP result when the operation succeeds. Must return TSuccess.

logger ILogger

An optional logger for logging failure cases. If null, no logging occurs.

Returns

Results<TSuccess, TUnauthorized, TBadRequest, TProblem>

A Microsoft.AspNetCore.Http.HttpResults.Results<TResult1, TResult2, TResult3, TResult4, TResult5> representing the HTTP response.

Type Parameters

TSuccess

The type of the successful HTTP result (e.g., Microsoft.AspNetCore.Http.HttpResults.Ok<TValue>, Microsoft.AspNetCore.Http.HttpResults.NoContent, Microsoft.AspNetCore.Http.HttpResults.Created<TValue>).

TUnauthorized

The type of the unauthorized HTTP result (e.g., Microsoft.AspNetCore.Http.HttpResults.UnauthorizedHttpResult).

TBadRequest

The type of the bad request HTTP result (e.g., Microsoft.AspNetCore.Http.HttpResults.BadRequest).

TProblem

The type of the problem HTTP result (e.g., Microsoft.AspNetCore.Http.HttpResults.ProblemHttpResult).

Examples

Usage for a generic result (e.g., GET operation):

var response = await mediator.Send(new AssetFindOneQuery(id), cancellationToken);
return MapResult{Ok, NotFound, UnauthorizedHttpResult, BadRequest, ProblemHttpResult}(
    response.Result,
    logger,
    cancellationToken);

Usage for a non-generic result (e.g., DELETE operation):

var response = await mediator.Send(new AssetDeleteCommand(id), cancellationToken);
return MapResult{NoContent, NotFound, UnauthorizedHttpResult, BadRequest, ProblemHttpResult}(
    response.Result,
    _ => TypedResults.NoContent(),
    logger,
    cancellationToken);

Usage for a create operation:

var response = await mediator.Send(new AssetCreateCommand(asset), cancellationToken);
return MapResult{Created{Asset}, NotFound, UnauthorizedHttpResult, BadRequest, ProblemHttpResult}(
    response.Result,
    value => TypedResults.Created($"/api/core/assets/{value.Id}", value),
    logger,
    cancellationToken);

Example response for success: HTTP 200 OK, 204 No Content, or 201 Created (depending on TSuccess). Example response for unauthorized: HTTP 401 Unauthorized. Example response for not found: HTTP 404 Not Found. Example response for bad request: HTTP 400 Bad Request. Example response for other errors: HTTP 400 Problem with details.

Exceptions

ArgumentNullException

Thrown when result or successFunc is null.

RegisterErrorHandler<TError>(Func<ILogger, Result, IResult>)

Registers a custom error handler for a specific error type.

public static void RegisterErrorHandler<TError>(Func<ILogger, Result, IResult> handler) where TError : IResultError

Parameters

handler Func<ILogger, Result, IResult>

A function that takes a logger and result and returns an IResult.

Type Parameters

TError

The type of error to register a handler for.

Examples

ResultMapHttpExtensions.RegisterErrorHandler<CustomBusinessError>((logger, result) => {
    logger?.LogWarning("Business rule violation: {Error}", result.ToString());
    return TypedResults.Problem(
        detail: result.ToString(),
        statusCode: 422,
        title: "Business Rule Violation",
        type: "https://example.com/errors/business-rule");
});

Remarks

If a handler is already registered for the error type, it will be replaced (last registration wins). This handler will be invoked before the default error handling logic when the Result contains an error of type TError.
